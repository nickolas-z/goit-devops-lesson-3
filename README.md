# [Lesson 3 — ML Model Containerization](https://www.edu.goit.global/learn/25315460/38664777/38699480/homework)

DevOps practice: ML model containerization (MobileNetV2 TorchScript inference)
with fat vs slim Docker image comparison.

## Project Structure

```text
lesson-3/
├── .dockerignore         # Excludes unnecessary files from build context
├── compare.sh            # One script that runs the comparison workflow end to end

├── Dockerfile.fat        # Fat image: python:3.9 + all tools

├── install_dev_tools.sh  # Idempotent environment setup bash script
├── model.pt              # TorchScript model — generated by export_model.py
├── test.jpg              # Sample image for local/Docker inference checks

```

## Quick Start

### 1. Setup Environment

```bash
chmod +x install_dev_tools.sh
sudo ./install_dev_tools.sh
# Check installation details:
cat install.log
```

Script installs: Docker + Compose v2, Python 3.9+, pip3, torch, torchvision, pillow, django.
Idempotent — safe to re-run.

### 2. Run Comparison Script

```bash
bash compare.sh
```

The script gathers the commands from this README into one place: export if
`model.pt` is missing, build, compare sizes, count layers, run Docker inference,
and inspect metadata. Set `FORCE_EXPORT=1` to regenerate `model.pt`.

### 3. Export Model Manually

```bash
# Activate venv
source .venv/bin/activate

python export_model.py
# Creates model.pt (~14 MB)
```

**Note:** model.pt is required before `docker build`. If venv is not activated — run via `.venv/bin/python export_model.py`.

### 4. Run Inference Locally

```bash
# Activate venv (if not already active)
source .venv/bin/activate

# Single image (positional)
python inference.py path/to/image.jpg

# Single image (flag)
python inference.py --image path/to/image.jpg

# All images in a folder
python inference.py path/to/folder/

# Without venv activation:
.venv/bin/python inference.py path/to/image.jpg
```

Example output for the current `test.jpg`:

```text
[info] Loaded 1000 ImageNet class labels from local file.

Top-3 predictions:
  #1 beacon (437): 99.25%
  #2 breakwater (460): 0.16%
  #3 promontory (976): 0.03%
```

### 5. Build Docker Images

```bash
# Fat image (~12.4 GB, CUDA torch + all dev tools — NOT for production)
docker build -f Dockerfile.fat -t lesson3-fat .

# Slim image (~1.02 GB, multi-stage, CPU-only torch)
docker build -f Dockerfile.slim -t lesson3-slim .
```

### 6. Run with Docker

```bash
# Requires a test image file
docker run --rm -v "$(pwd)/test.jpg:/app/test.jpg" lesson3-fat  /app/test.jpg
docker run --rm -v "$(pwd)/test.jpg:/app/test.jpg" lesson3-slim /app/test.jpg
```

Both images return the same prediction for `test.jpg`:

```text
[info] Loaded 1000 ImageNet class labels from local file.

Top-3 predictions:
  #1 beacon (437): 99.25%
  #2 breakwater (460): 0.16%
  #3 promontory (976): 0.03%
```

### 7. Compare Image Sizes

```bash
# Image sizes
docker images | grep lesson3

# Layer count
docker inspect lesson3-fat  | python3 -c \
  "import sys,json; d=json.load(sys.stdin); print('fat layers:',  len(d[0]['RootFS']['Layers']))"
docker inspect lesson3-slim | python3 -c \
  "import sys,json; d=json.load(sys.stdin); print('slim layers:', len(d[0]['RootFS']['Layers']))"
```

Measured result:

```text
lesson3-fat   latest  2a3e39d8b1a9  About a minute ago  12.4GB
lesson3-slim  latest  11d09be77f36  22 minutes ago      1.02GB

fat layers: 11
slim layers: 46
```

## Key Differences: Fat vs Slim

| Property | Fat | Slim |
| --- | --- | --- |
| Base image | python:3.9 | builder: python:3.11-slim-bookworm, runtime: distroless/python3-debian12 (pinned digest) |
| Multi-stage build | No | Yes (builder + runtime) |
| Torch variant | 2.8.0+cu128 (~6 GB) | 2.11.0+cpu / torchvision 0.26.0+cpu (~1 GB) |
| Image size / layers | 12.4 GB / 11 layers | 1.02 GB / 46 layers |
| Dev tools | wget, curl, vim, git | Absent |
| `--no-install-recommends` | No | Yes |
| Purpose | Learning / debugging | Closer to production |

Detailed analysis: [report.md](report.md)

## Requirements

- For production, conversion to ONNX Runtime is recommended (see `report.md`).
